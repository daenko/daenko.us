HTML5

The HTML standard has been evolving to meet the demands of viewers and web developers HTML5 is the most recent update. There are a number of new features and several modifications that are applicable to multimedia on the web. Review one or more HTML5 Introductions and tutorials. Check out movies and games that are created entirely in HTML5. What are some of the big changes? Why are these important to you as a web developer?

The questions about HTML5 updates are: why is there so much excitement around it and why does it matter for me as for user and future web developer? I find this article very good at answering these questions. I'd like to share these HTML5 features and principles:

  • New semantic elements like header, footer, article, and section
  • New form control attributes like number, date, time, calendar, and range
  • New graphic elements: svg and canvas. The canvas tag allows the programmer to build graphics interactively
  • New multimedia elements: audio and video. The developers do not need to use plugins, becausese they can directly embed audio and video in the webpage
  • Also HTML5 provides a lot of new APIs: HTML Geolocation, HTML Drag and Drop, HTML Local Storage, HTML Application Cache, HTML Web Workers, HTML SSE
  • Less dependence on plugins for functionality.
  • Scripting should be replaced with markup whenever possible.
  • Device independence (i.e., available on all devices and providing the same end experience).
  • Public development process so people can see what’s going on.

HTMLl5 gives me as the user and as the web developer the following advantages:

  • web pages should load faster
  • less bandwidth should be used
  • battery life on mobile devices could last longer
  • I won’t have to deal with different cases in web design since the same results display on multiple browsers
  • HTML5 is more organized than and the semantics have improved, making code more readable

Preview on Mobile devices

Use the Thimble mobile view or ipadpeek.com to view your page. What do you think? Have you created a good viewing experience for someone using a mobile device to access your page? How important is it to accommodate mobile users? What could you do differently to improve mobile viewing?

It is the primary purpose of usability to make a web page mobile friendly. I've read about "Mobile First" approach in the book by Luke Wroblewski, who tells that mobile version is the primary and is designed before the desktop one. I tested my page using mobile-friendly test from google, and I found out that it is not mobile-friendly, because of too small text, mobile viewport not set, links too close together. I think I might use separate css for mobile devices by adding media queries.
